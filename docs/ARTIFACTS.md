# Artifact Generation

Open Notebook can turn research notebooks into 14 distinct artifact types — slide decks, infographics, mind maps, videos, and more — all generated by real LLM calls with structured output and fully-traceable provenance.

The design target is **NotebookLM parity**: one platform, one pipeline, any output format.

## How it works

```
┌───────────────────────┐   ┌───────────────────────┐   ┌────────────────────────┐
│  ArtifactRequest      │──▶│  chunked_generate()   │──▶│  Real LLM call         │
│  (sources, config)    │   │  map-reduce if long   │   │  (structured output)   │
└───────────────────────┘   └───────────────────────┘   └───────────┬────────────┘
                                                                    │
                                              ┌─────────────────────┘
                                              ▼
                                ┌───────────────────────────┐
                                │  Validated Pydantic schema │
                                │  (SlideDeckSchema, …)      │
                                └──────────────┬────────────┘
                                               │
                                               ▼
                                ┌───────────────────────────┐
                                │  Renderer stack            │
                                │  (pptx / docx / svg / mp4) │
                                └──────────────┬────────────┘
                                               │
                                               ▼
                                ┌───────────────────────────┐
                                │  ArtifactResult            │
                                │  + GenerationProvenance    │
                                └───────────────────────────┘
```

1. A user submits `POST /api/artifacts/generate` with a notebook_id or inline sources.
2. The API service submits a job to the `surreal-commands` queue (`generate_artifact` command, `max_attempts=5`, exponential jitter, `stop_on=[ValueError]`) and returns `{job_id}` immediately.
3. The queue worker pulls the job, calls the generator class (e.g. `SlideDeckGenerator`), which calls `ArtifactLLM.generate_structured(prompt, SlideDeckSchema)`.
4. Structured output is dispatched provider-natively: OpenAI json_schema strict, Anthropic tool-use, Gemini response_schema, OpenRouter pass-through.
5. Long inputs flow through `chunked_generate()` — per-chunk extraction, then a final synthesis pass — instead of being truncated.
6. The validated Pydantic payload is handed to the renderer stack (themed pptx, SVG infographic, MP4 video, etc.).
7. Every LLM call is recorded in `ArtifactResult.provenance` (provider, model, tokens, latency, attempt) and streamed to loguru as structured JSON.

Poll `GET /api/artifacts/jobs/{job_id}` to see status; when complete the response carries `files`, `structured`, and `provenance`.

## Using it from the frontend

Open the **Artifacts** page in the sidebar. Click **New Artifact**, pick one of the 14 types, choose a notebook or paste inline content, optionally tweak config (max items, title), and hit Generate. The UI polls the job every 2 seconds, shows elapsed time and current status, and once complete lets you preview the output inline (markdown, PNG, video, HTML) or download the binary file. A collapsible **Provenance** panel shows the provider, model, and per-call token/latency counts — proof that a real LLM served the request.

## Using it from the API

```bash
# Submit a job
curl -X POST http://localhost:5055/api/artifacts/generate \
  -H "Content-Type: application/json" \
  -d '{
    "artifact_type": "slide_deck",
    "notebook_id": "notebook:abc123",
    "title": "Q4 Research Digest",
    "config": {"max_items": 10}
  }'
# → {"job_id": "command:xyz789", "status": "submitted"}

# Poll status
curl http://localhost:5055/api/artifacts/jobs/command:xyz789
# → {"status":"completed","artifact_type":"slide_deck","title":"Q4 Research Digest","files":[{"path":"/data/artifacts/q4_research_digest_abc123.pptx","mime_type":"application/vnd.openxmlformats-officedocument.presentationml.presentation","description":"Slide deck (PPTX)"}],"provenance":{"provider":"anthropic","model":"claude-sonnet-4-6","calls":[{"tokens_in":2108,"tokens_out":3492,"latency_ms":12430,"attempt":1,"status":"ok"}]}, ...}

# Download the binary
curl "http://localhost:5055/api/artifacts/download?path=/data/artifacts/q4_research_digest_abc123.pptx" > deck.pptx
```

The download endpoint is path-traversal guarded: it refuses anything outside `ARTIFACT_OUTPUT_ROOT` with HTTP 403.

## Configuration

Minimum requirement: a provider key for at least one LLM, configured either via the UI (Settings → API Keys) or env vars.

| Env var | Purpose |
|---|---|
| `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` / `GOOGLE_API_KEY` / `OPENROUTER_API_KEY` / `XAI_API_KEY` | At least one required. Routed via Esperanto. |
| `ELEVENLABS_API_KEY` | Optional — TTS provider for `video_overview` if you prefer ElevenLabs over OpenAI `tts-1-hd`. |
| `ARTIFACT_OUTPUT_ROOT` | Optional. Defaults to `./data/artifacts`. Every generated file lives under this root; the download endpoint enforces containment. |
| `ARTIFACT_RENDER_VIDEO` | Set to `1` to render the MP4 on `video_overview` generation. Off by default because ffmpeg is an extra deploy prerequisite. |
| `ARTIFACT_RENDER_STUB` | Dev-only short-circuit for the video renderer (writes a 1-second silent MP4 with script metadata). Rejected in production (`ENV=production`). |

Native libraries needed for full renderer coverage:

- **cairo** (for SVG infographics): `brew install cairo` on macOS, `apt-get install libcairo2` on Debian/Ubuntu. Tests auto-skip when absent.
- **ffmpeg** (for video overview): `brew install ffmpeg` or `apt-get install ffmpeg`.
- **graphviz** (optional fallback for mindmap): `brew install graphviz` or `apt-get install graphviz`. Mermaid via `npx @mermaid-js/mermaid-cli` is the primary path and only needs Node.

## What's intentionally NOT in this module

- **Heuristic fallbacks.** There are none. If no LLM provider is configured, generation raises `ConfigurationError` (HTTP 422) rather than serving keyword-ranked template text. A CI lint (`tests/artifacts/unit/test_no_heuristic.py`) fails the build if any `heuristic_json` / `_heuristic_text` / `heuristic.py` reference reappears.
- **Synchronous HTTP generation.** The API only exposes the queue-based flow. A long-running LLM call never ties up a FastAPI worker.
- **Silent truncation of long inputs.** `ArtifactRequest.combined_content()` returns full fidelity; long content flows through `chunked_generate()` (map + reduce).

## Provenance — the "real LLM" guarantee

Every `ArtifactResult` carries:

```python
provenance: GenerationProvenance {
    run_id: str
    artifact_type: str
    model_id: str
    calls: List[LLMCallRecord {
        prompt_hash: str
        tokens_in: int
        tokens_out: int
        latency_ms: int
        attempt: int
        provider: str  # anthropic / openai / openrouter / google / xai
        model: str
        status: "ok" | "error" | "config_error"
        error_class: Optional[str]
    }]
}
```

Tests assert:

1. **Dead-code lint** — no `heuristic_json` or `_heuristic_text` references in `open_notebook/artifacts/**/*.py`; no `ARTIFACT_USE_LLM` env gate.
2. **Call counter** — integration tests that hit real providers assert `len(provenance.calls) >= 1` and `provider != "injected"`.
3. **Cross-provider matrix** — `tests/artifacts/integration/test_matrix.py` runs 3 artifact types × 3 providers (Anthropic + OpenAI + OpenRouter); all 9 cells must pass before merge.

## Adding a new artifact type

1. Create `open_notebook/artifacts/generators/my_artifact.py`:

   ```python
   from pydantic import BaseModel, Field

   from open_notebook.artifacts.base import (
       ArtifactRequest, ArtifactResult, BaseArtifactGenerator,
   )
   from open_notebook.artifacts.registry import register_generator


   class MyArtifactSchema(BaseModel):
       title: str = Field(..., description="...")
       sections: list[str] = Field(..., description="...")


   @register_generator
   class MyArtifactGenerator(BaseArtifactGenerator):
       artifact_type = "my_artifact"
       description = "Plain-English description of what this produces."
       default_model_type = "transformation"

       async def generate(self, request: ArtifactRequest) -> ArtifactResult:
           def map_prompt(chunk: str) -> str:
               return f"Extract key points from:\n\n{chunk}"

           def reduce_prompt(partials: list[MyArtifactSchema]) -> str:
               return f"Synthesise: {[p.model_dump() for p in partials]}"

           result = await self.chunked_generate(
               request,
               schema=MyArtifactSchema,
               map_prompt_builder=map_prompt,
               reduce_prompt_builder=reduce_prompt,
           )

           # … render to disk …

           return ArtifactResult(
               artifact_type=self.artifact_type,
               title=result.title,
               structured=result.model_dump(),
               files=[...],
               provenance=self.llm.provenance,
           )
   ```

2. Register it in `open_notebook/artifacts/registry.py` inside the appropriate sentinel region (`# <BATCH B>` or `# <BATCH C>`).
3. Add a unit test at `tests/artifacts/unit/test_gen_my_artifact_schema.py`.
4. If the output isn't Markdown/JSON, add a renderer module and wire it up.

## Troubleshooting

- **`ConfigurationError: No model configured for default_type='transformation'`** — Set a default transformation model in Settings → Models, or provide `model_id` in the request body.
- **`ExternalServiceError: LLM call failed ... invalid_api_key`** — The configured provider key is wrong or expired. Test via Settings → API Keys → Test.
- **Video render fails with "ffmpeg not found"** — Install ffmpeg on the host, or set `ARTIFACT_RENDER_STUB=1` for dev-only 1-second silent output.
- **SVG infographic silently falls back to Pillow** — Install `libcairo` (`brew install cairo` / `apt-get install libcairo2`). The SVG pipeline produces substantially higher quality than the bitmap fallback.
- **HTTP 403 on /artifacts/download** — The path is outside `ARTIFACT_OUTPUT_ROOT`. This is correct behaviour; generated files always sit inside that root.
